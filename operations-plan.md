# Operations Plan

作成日: 2026-06-13

## 基本方針

最初からSNS全自動収集を狙わない。初期運用は「人間が根拠URLを集め、AIが整理し、人間が確認して公開する」半自動型にする。

Substackは読者獲得と有料レポート配信に使い、独自Webサイトは論点ページや比較表を見せる場所として育てる。

## 運用全体像

| 工程 | 内容 | 担当 |
| --- | --- | --- |
| テーマ選定 | 今週扱う論点を選ぶ | 編集者 |
| URL収集 | 報道、公式資料、本人発言、代表的SNS投稿を集める | リサーチャー |
| AI整理 | 要旨、事実、未確認、賛否、論点、時系列を抽出 | AI |
| 人間レビュー | 根拠確認、表現修正、リスクチェック | 編集者 |
| Web公開 | 論点ページとして公開 | 編集者/開発 |
| Substack配信 | 無料版と有料版を配信 | 編集者 |
| 更新 | 反論、追加報道、訂正を反映 | 編集者 |

## 週間運用

| 曜日 | 作業 |
| --- | --- |
| 月 | 今週の候補テーマを3-5本選定 |
| 火 | 公式資料、報道、本人発言を収集 |
| 水 | AIで初稿生成。論点、賛否、未確認情報を抽出 |
| 木 | 人間レビュー。根拠不足、名誉毀損、偏りを修正 |
| 金 | Web公開、Substack無料版配信 |
| 土 | 有料版の深掘り、比較表、補足資料を配信 |
| 日 | 反応確認、翌週テーマ候補を整理 |

## Substack運用

### 役割

Substackは、収益化の入口として使う。

| 用途 | 内容 |
| --- | --- |
| 無料ニュースレター | 週1回、注目論点の簡易整理を配信 |
| 有料記事 | 深掘り分析、根拠一覧、比較表、時系列を配信 |
| 読者獲得 | メールリストを蓄積する |
| 信頼形成 | 定期配信で編集方針と品質を見せる |

### 無料版の内容

- 今週の論点3本
- 各論点の30秒要約
- 賛成・反対の代表意見
- 根拠リンクの一部
- 独自Webページへの導線

### 有料版の内容

- 詳細な時系列
- 政党・議員ごとの立場比較
- SNS反応の分類
- 根拠リンク全文リスト
- 反論マップ
- 過去類似事例との比較
- PDFまたはCSVの簡易資料

### 価格案

| プラン | 価格 | 内容 |
| --- | --- | --- |
| Free | 0円 | 簡易要約、代表的賛否、根拠リンク一部 |
| Paid | 500-980円/月 | 詳細分析、時系列、根拠一覧、比較表 |
| Annual | 5,000-9,800円/年 | 月額より割安 |
| Business | 30,000円/月〜 | カスタム調査、月次レポート、資料化 |

## 独自Webサイト運用

### 役割

独自Webサイトは、Substackでは表現しにくい構造化ページを表示する。

| ページ | 内容 |
| --- | --- |
| 事例ページ | 1つの論点を整理 |
| 比較ページ | 政党、議員、人物、主張を横並び比較 |
| 一覧ページ | 話題の論点を探す |
| 根拠ページ | 参照元、発言、資料を蓄積 |

### 初期構成

最初はDBではなく、JSONデータでよい。

```text
data/
  issues.json
  sources.json
  actors.json
  stances.json
```

将来的にPostgreSQLへ移行する。

## 初期テーマ別の運用

### 議員定数削減

MVPの最初の題材に向いている。

収集するもの:

- 政党公式見解
- 国会発言
- 選挙公約
- 議員本人投稿
- 議員数、報酬、財政効果、他国比較

表示する論点:

- 税金削減
- 身を切る改革
- 代表制
- 地方の声
- 少数政党への影響
- 国会の行政監視機能

### 憲法改正論議

政党・議員スタンスDBに向いている。

収集するもの:

- 党公式資料
- 国会会議録
- 憲法審査会発言
- 選挙公約
- 議員公式サイト

表示する論点:

- 9条
- 自衛隊明記
- 緊急事態条項
- 教育無償化
- 参院合区
- 国民投票法

注意:

「改憲賛成/反対」の二択にしない。必ず論点単位で表示する。

### 辺野古高校生事故

事故・未成年・被害者保護の観点から慎重運用が必要。

収集するもの:

- 警察・自治体・学校・報道で確認できる事実
- 政治家や団体の公式コメント
- 代表的なSNS反応

表示する論点:

- 事故原因
- 抗議活動との関係
- 基地問題との関係
- 政治利用への批判
- 被害者保護

禁止・制限:

- 氏名、顔写真、学校名、家族情報の掲載
- 未確認の責任断定
- 被害者や関係者への攻撃的表現の引用拡散

## 品質管理

公開前チェックリスト:

- 事実と意見が分離されているか
- 未確認情報にラベルが付いているか
- 根拠URLがあるか
- 本人否定や反論がある場合に掲載しているか
- 個人攻撃や差別表現を増幅していないか
- 見出しが煽りすぎていないか
- 訂正可能な構造になっているか

## 収益化ロードマップ

| フェーズ | 目標 |
| --- | --- |
| Phase 1 | 無料Webページと無料Substackで読者を集める |
| Phase 2 | 有料Substackで週1本の深掘りを出す |
| Phase 3 | 議員・政党比較など有料向け機能を追加 |
| Phase 4 | B2B向け月次レポートを販売 |
| Phase 5 | 独自Webサイトに会員機能とDB検索を実装 |

## KPI

| KPI | 目的 |
| --- | --- |
| 無料購読者数 | 読者基盤の成長 |
| 有料転換率 | 収益性 |
| 開封率 | テーマ選定と信頼度 |
| クリック率 | 根拠リンク・Webページへの関心 |
| 継続率 | 有料価値の検証 |
| 訂正件数 | 品質管理 |
| B2B問い合わせ数 | 高単価化の可能性 |
