# テーマ別分類設計 v2

## 目的

Yahooリアルタイム検索で取得したSNS反応を、テーマごとの投稿傾向に合わせて分類する。
従来の「賛成/反対」中心の一律分類では、生活・教育系では保留が多くなり、政治系では引用、皮肉、人物批判、政策賛否の混同が起きやすい。

v2では、テーマを型に分けて分類スキーマを変える。

## 担当

- 主担当: Claude Code
- 理由: 最終的な新規テーマ作成、`configs/topics/*.yaml` 作成、Yahoo収集、Ollama分類、HTML生成までをClaude Codeが担うため。
- レビュー担当: Hermes
- Hermesの役割: カテゴリ粒度、読者への分かりやすさ、UI表示軸、記事化しやすさをレビューする。

## 共通原則

- 件数は世論比率ではなく、検索サンプル内の論点分布として扱う。
- 単なるニュース共有、URL紹介、意味不明、対象外投稿は無理に賛否へ寄せない。
- 代表投稿として表示・記事利用するものは `confidence >= 0.75`、`risk != high`、`article_usable: true` を優先する。
- 攻撃的表現や未確認断定は、分類はしても要約で中和し、必要に応じて `risk: high` または `review_required: true` にする。
- クエリ設計は分類設計の一部として扱う。分類不能な投稿が多いテーマは、モデル調整より先に検索語を見直す。

## テーマ型

### A. 政治・政策論争型

対象例:

- `constitutional`
- `takaichi`
- 今後の国会、政党、選挙、制度改革系

問題:

- 引用元への反発と、引用元の主張への反発が混ざる。
- 人物・政党批判と政策賛否が混ざる。
- 皮肉、反語、党派ラベルによる誤読が起きやすい。
- 1投稿内に複数争点が入る。

推奨スキーマ:

```json
{
  "category": "最終カテゴリ",
  "stance": "表示用の大分類",
  "issue": "主論点",
  "topic_target": "大きな話題",
  "actor_target": "評価対象の主体",
  "criticized_target": "実際に批判されている対象",
  "stance_to_target": "評価する / 批判する / 慎重 / 不明",
  "stance_to_quoted_author": "賛成 / 反対 / 中立 / 不明",
  "stance_to_quoted_claim": "賛成 / 反対 / 中立 / 不明",
  "confidence": 0.0,
  "confidence_level": "high / medium / low",
  "alternate_categories": [],
  "review_required": false,
  "summary": "中立的な1文要約",
  "reason": "分類理由",
  "article_usable": true,
  "risk": "none / low / medium / high"
}
```

運用:

- `category` はUI表示と集計用。
- `issue` は論点マップ用。
- `criticized_target` と `stance_to_quoted_claim` を必須に近い扱いにする。
- `confidence_level: low` は集計には含めても、代表投稿表示では優先度を下げる。

### B. 事故・炎上・複合対象型

対象例:

- `henoko`
- 事故、学校対応、行政判断、抗議活動、報道批判が混ざるテーマ

問題:

- 事故そのもの、学校、行政、報道、政治運動への反応が混ざる。
- 追悼・安全管理・政治的中立性・基地問題が同時に出る。
- 「誰を批判しているのか」を外すとカテゴリが崩れる。

推奨スキーマ:

- 政治・政策論争型と同じ構造化スキーマを使う。
- `actor_target` と `criticized_target` を特に重視する。
- `issue` は「追悼・慰霊」「安全管理」「教育基本法・政治的中立性」「政治利用」「報道・行政対応」「基地問題」「未確認」のように分ける。

運用:

- 既存の `scripts/classify_henoko_structured_ollama_batch.py` を基準にする。
- 旧 `henoko.yaml` の単純分類は、互換用または軽量分類用に限定する。

### C. 生活・教育・マナー型

対象例:

- `school-nickname-ban`
- 校則、教育現場、日常マナー、呼称、働き方など

問題:

- 明確な賛成/反対が出にくい。
- 体験談、雑談、ニュース共有が多い。
- 「条件付き」「違和感」「ケースによる」が多く、二分法では保留が増える。

推奨スキーマ:

```json
{
  "category": "論点カテゴリ",
  "stance": "支持 / 懸念 / 条件付き / 体験談 / 情報共有 / 不明",
  "issue": "いじめ防止 / 心理的安全 / ジェンダー配慮 / 親密さ・呼称文化 / 一律運用批判 / 現場裁量 / 体験談 / 情報共有",
  "policy_attitude": "肯定 / 否定 / 条件付き / 判断不能",
  "experience_type": "本人経験 / 保護者視点 / 教員視点 / 一般論 / なし",
  "confidence": 0.0,
  "confidence_level": "high / medium / low",
  "summary": "中立的な1文要約",
  "reason": "分類理由",
  "article_usable": true,
  "risk": "none / low / medium / high"
}
```

カテゴリ案:

- いじめ防止・心理的安全を重視
- ジェンダー配慮・一律さん付けを支持
- 親密さや呼称文化が失われる懸念
- 一律禁止は表面的・実効性が低いという批判
- 嫌な呼称だけ止めるなど柔軟運用を支持
- 体験談・現場感覚の共有
- ニュース共有・立場不明
- 対象外・分類保留

運用:

- UIでは「賛成/反対」より「どの論点に反応しているか」を前面に出す。
- `policy_attitude` は補助軸として使う。
- 分類率が低い場合は、`あだ名禁止 やりすぎ`、`さん付け 指導 違和感`、`あだ名禁止 いじめ防止` のような意見語入りクエリを増やす。

### D. 技術・権利・産業対立型

対象例:

- `ai-copyright`

問題:

- 法規制、倫理、実用、産業競争力、クリエイター被害が混ざる。
- 「AIに反対」なのか「無断学習に反対」なのかが違う。
- 利用者批判と制度批判が混ざる。

推奨スキーマ:

```json
{
  "category": "論点カテゴリ",
  "stance": "規制強化 / 規制慎重 / 条件付き容認 / モラル重視 / 実用重視 / 不明",
  "issue": "著作権 / 無断学習 / ライセンス / 対価還元 / クレジット表記 / 産業競争力 / 利便性 / 不明",
  "target": "AI開発者 / AI利用者 / 権利者 / 政府・制度 / 規制派 / 不明",
  "policy_attitude": "強化支持 / 慎重 / 条件付き / 判断不能",
  "confidence": 0.0,
  "confidence_level": "high / medium / low",
  "summary": "中立的な1文要約",
  "reason": "分類理由",
  "article_usable": true,
  "risk": "none / low / medium / high"
}
```

運用:

- 既存カテゴリは比較的よいが、`issue` と `target` を追加すると誤分類検出がしやすい。
- 「AI利用者のモラル批判」を「規制賛成」に寄せすぎない。

### E. 制度移行・公共サービス型

対象例:

- `bukatsu-chiiki`
- 教育、福祉、行政サービス、地域移行、費用負担など

問題:

- 方向性には賛成だが実装に反対、という投稿が多い。
- 受け皿、費用、地域差、教員負担などの実務論点が多い。
- 単純賛否より「何を条件にするか」が重要。

推奨スキーマ:

```json
{
  "category": "論点カテゴリ",
  "stance": "支持 / 反対 / 条件付き / 懸念 / 情報共有 / 不明",
  "issue": "教員負担 / 少子化 / 費用負担 / 受け皿不足 / 地域差 / 教育的価値 / 段階導入 / 情報共有",
  "implementation_attitude": "方向性支持・実装懸念 / 方向性反対 / 条件付き支持 / 判断不能",
  "affected_actor": "教員 / 生徒 / 保護者 / 地域団体 / 学校 / 不明",
  "confidence": 0.0,
  "confidence_level": "high / medium / low",
  "summary": "中立的な1文要約",
  "reason": "分類理由",
  "article_usable": true,
  "risk": "none / low / medium / high"
}
```

運用:

- 既存カテゴリはおおむね維持できる。
- `implementation_attitude` を追加し、「方向性は賛成だが現状は無理」を表現できるようにする。

## 既存トピックへの適用方針

| トピック | 現状 | v2方針 |
| --- | --- | --- |
| `constitutional` | 構造化v2あり。保留が少なく良好 | 構造化v2を標準化。`criticized_target` と引用態度を追加検討 |
| `henoko` | 構造化分類あり。旧YAMLも残る | 構造化v2を主系統にする |
| `takaichi` | 単純分類。人物・報道・政策が混ざる | 政治・炎上型の構造化v2へ移行 |
| `school-nickname-ban` | v2でも保留が多い | 生活・教育型に再設計。賛否より論点分類を主にする |
| `ai-copyright` | 比較的分類しやすい | 技術・権利型として `issue` と `target` を追加 |
| `bukatsu-chiiki` | 比較的分類しやすい | 制度移行型として実装態度と影響主体を追加 |

## Hermesレビュー観点

- テーマ型の分け方が妥当か。
- UI表示に必要なカテゴリ粒度として過不足がないか。
- `stance`、`issue`、`target` 系フィールドの責務が重複していないか。
- Yahooリアルタイム検索で実際に拾える投稿に対して、過剰に細かすぎないか。
- 記事化・反応マップ化したとき、読者に伝わる軸になっているか。
- 既存HTML生成・SQLite取込・ポータル表示への影響が許容範囲か。
