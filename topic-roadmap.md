# 初期テーマ設計ロードマップ

作成日: 2026-06-13

## 方針

このサービスは、すべての論争を同じ形式で扱わない。テーマごとに「集めるべきデータ」「賛否の出方」「根拠の種類」が違うため、初期テーマを3タイプに分ける。

1. 事件・事故への反応型
2. 政策・憲法への立場表明型
3. 制度改革への利害対立型

## 1. 辺野古高校生事故に対する反応

### テーマ種別

事件・事故への反応型。

### 目的

事故そのものの事実関係、被害者・関係者への配慮、政治利用への批判、基地問題・抗議活動への評価が混ざりやすい。これらを分けて表示する。

### 画面で見せるべき項目

| 項目 | 内容 |
| --- | --- |
| 事故の要旨 | いつ、どこで、何が起きたか |
| 確認済み事実 | 警察、自治体、学校、報道で確認できる事実 |
| 未確認情報 | SNS上の推測、責任の断定、映像・写真の真偽 |
| 主要論点 | 事故原因、抗議活動との関係、基地問題、報道姿勢、政治家の反応 |
| 擁護・同情側 | 被害者保護、事故を政治利用すべきでない、事実確認が先 |
| 批判側 | 危険な行動・環境への批判、基地や抗議活動への批判、行政責任の追及 |
| 炎上リスク | 被害者特定、未成年者情報、誹謗中傷、政治的利用 |

### 向いている機能

- 時系列整理
- 事実/推測/意見の分離
- 被害者保護ラベル
- SNS反応の感情分類
- 「政治論点」と「事故原因」を分ける表示

### 注意点

未成年や事故被害者が関係する可能性があるため、個人情報、学校名、顔写真、氏名、家族情報は扱わない。政治的論争よりも安全・配慮を優先する。

## 2. 憲法改正論議の反応

### テーマ種別

政策・憲法への立場表明型。

### 目的

憲法改正は、SNSの感情的反応よりも、国会議員、政党、会派、専門家の公式立場を整理する価値が高い。党や議員ごとに「どの条文・論点に賛成/反対なのか」を見せる。

### 画面で見せるべき項目

| 項目 | 内容 |
| --- | --- |
| 論点 | 9条、自衛隊明記、緊急事態条項、教育無償化、参院合区、国民投票法など |
| 政党スタンス | 各党の公式見解、選挙公約、代表質問、会見 |
| 議員スタンス | 国会発言、採決行動、公式サイト、SNS投稿 |
| 賛成意見 | 安全保障、災害対応、時代変化への対応など |
| 反対意見 | 権力制限の弱体化、9条改変への懸念、緊急事態条項への警戒など |
| 中間意見 | 改正自体は否定しないが内容・手続きに慎重 |
| 根拠 | 国会会議録、党公式文書、選挙公約、議員本人投稿 |

### 向いている機能

- 政党別マトリクス
- 議員別スタンス検索
- 論点別賛否ヒートマップ
- 過去発言との変化検出
- 公約と実際の発言比較

### 例: 表示形式

| 論点 | 自民 | 公明 | 維新 | 国民民主 | 立憲 | 共産 |
| --- | --- | --- | --- | --- | --- | --- |
| 9条 | 要整理 | 要確認 | 要整理 | 要確認 | 慎重/反対寄り | 反対 |
| 緊急事態条項 | 要整理 | 要確認 | 要整理 | 要確認 | 慎重/反対寄り | 反対 |
| 教育無償化 | 要整理 | 要確認 | 要整理 | 要確認 | 要確認 | 要確認 |

実データ投入時は、必ず党公式資料・国会会議録・議員本人発言で埋める。

### 注意点

憲法論議は「改憲賛成/反対」の二択にすると雑になる。必ず条文・論点単位に分ける。

## 3. 議員定数削減

### テーマ種別

制度改革への利害対立型。

### 目的

議員定数削減は「身を切る改革」として支持されやすい一方、代表制の弱体化、少数政党・地方の声の排除、国会機能低下への懸念がある。ポピュリズム的賛成と制度論的反対が衝突しやすいテーマとして扱う。

### 画面で見せるべき項目

| 項目 | 内容 |
| --- | --- |
| 要旨 | どの議会、どの定数を、どれだけ削減する案か |
| 賛成意見 | 税金削減、政治家の身を切る改革、国民感覚との一致 |
| 反対意見 | 代表制低下、地方・少数派の声が届きにくくなる、行政監視力低下 |
| 政党スタンス | 各党の公約、法案、発言 |
| 制度論点 | 一票の格差、比例代表、選挙制度、国会審議時間 |
| 数字 | 議員報酬総額、削減効果、他国比較、人口あたり議員数 |
| 利害 | 大政党、小政党、地方、有権者、行政 |

### 向いている機能

- 数字で見る効果
- 賛成/反対の制度影響シミュレーション
- 「コスト削減」と「代表制」のトレードオフ表示
- 政党別スタンス比較

### 例: 論点整理

| 論点 | 賛成側 | 反対側 |
| --- | --- | --- |
| 税金 | 議員報酬・経費を削減できる | 財政効果は限定的 |
| 政治改革 | 政治家が身を切る姿勢を示せる | 人気取りで制度劣化を招く |
| 代表制 | 少数精鋭でよい | 地方・少数派の声が届きにくくなる |
| 国会機能 | 無駄な議員を減らせる | 委員会・審議・行政監視が弱くなる |

## MVPでの優先順位

| 優先度 | テーマ | 理由 |
| --- | --- | --- |
| 1 | 議員定数削減 | 賛否が明確で、制度論・数字・政党比較に落としやすい |
| 2 | 憲法改正論議 | 価値は高いが、論点が多く、公式資料の確認工数が大きい |
| 3 | 辺野古高校生事故 | 社会的関心は高いが、事故・未成年・個人情報への配慮が必要で運用難度が高い |

## データ収集方針

| データ種別 | 初期方針 |
| --- | --- |
| ニュース記事 | 手動URL登録から開始 |
| 公式資料 | 政党公式サイト、国会会議録、議員公式サイトを優先 |
| SNS投稿 | 代表的投稿URLを手動登録 |
| X全量取得 | MVPではやらない |
| AI要約 | URL本文をもとに、事実・未確認・賛成・反対・論点に分類 |
| 人間レビュー | 公開前に必須 |

## 初期プロダクト案

まずは「論点ページ作成ツール」として作る。

1. 管理者がURLを登録する。
2. AIが要旨、事実、未確認、賛成、反対、論点を抽出する。
3. 管理者が確認・修正する。
4. 公開ページに、比較表・賛否カード・根拠リンクを表示する。

SNS全自動収集ではなく、根拠URLベースの半自動整理から始める。
