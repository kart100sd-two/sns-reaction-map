## タスク: Supabaseプロジェクト作成＋投票テーブルのセットアップ

あなたは「SNS反応まっぷ」プロジェクトの投票バックエンド担当です。

### 背景

前回のタスクで、全5つの反応マップHTML（docs/*-reaction-map.html）にSupabase統合コードを追加済みです。しかし、Supabaseプロジェクト自体はまだ作成されていません。現在 `supabaseUrl=""` と `supabaseAnonKey=""` が空のため、localStorageフォールバックで動いています。

### 今回やること

Supabase（https://supabase.com）でプロジェクトを作成し、投票機能の本番接続に必要なものを準備してください。

#### 1. Supabaseプロジェクト作成
- プロジェクト名: `sns-reaction-map`
- リージョン: 東京（ap-northeast-1）推奨

#### 2. 投票テーブル作成（SQL）

以下のSQLをSupabaseのSQL Editorで実行してください：

```sql
CREATE TABLE votes (
  id BIGSERIAL PRIMARY KEY,
  topic_id TEXT NOT NULL,
  choice_idx INTEGER NOT NULL,
  ip_addr INET DEFAULT (current_setting('request.headers', true)::json->>'x-forwarded-for')::inet,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 24時間以内の同一IP・同一トピック重複投票を防止
CREATE UNIQUE INDEX votes_ip_topic_24h ON votes (topic_id, ip_addr, (created_at::date));

-- RLS（Row Level Security）有効化
ALTER TABLE votes ENABLE ROW LEVEL SECURITY;

-- 誰でもINSERTできる（匿名投票）
CREATE POLICY "Anyone can vote" ON votes
  FOR INSERT WITH CHECK (true);

-- 誰でもSELECTできる（集計結果を見る）
CREATE POLICY "Anyone can read votes" ON votes
  FOR SELECT USING (true);
```

#### 3. 接続情報の取得

以下の2つの値を取得して報告してください：
- **Supabase URL**: Settings > API > Project URL
- **Anon Key**: Settings > API > Project API keys > anon public

#### 4. 動作確認

Supabase管理画面のSQL Editorで、テスト投票を挿入して確認：

```sql
INSERT INTO votes (topic_id, choice_idx) VALUES ('test', 0);
SELECT * FROM votes;
DELETE FROM votes WHERE topic_id = 'test';
```

### 成果物

以下を出力してください：
1. Supabase URL
2. Anon Key（publicキーなので共有OK）
3. 作成したテーブルのスキーマ確認結果
4. テスト投票の成功確認

これらの情報をもらえれば、Claude Codeが各HTMLファイルの `supabaseUrl` と `supabaseAnonKey` に値を設定して本番接続を完了します。

### 注意
- **HTMLファイルの編集は不要です**（Claude Codeが行います）
- Supabaseの無料プランで十分です
- Service Role Keyは共有しないでください（Anon Keyのみ）
