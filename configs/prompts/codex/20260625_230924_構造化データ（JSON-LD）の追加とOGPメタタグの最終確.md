## タスク: 全ページに構造化データ（JSON-LD）を追加

あなたは「SNS反応まっぷ」プロジェクトのSEO担当です。

### 背景

サイトは GitHub Pages で公開済みです。
- 公開URL: https://issue-stance-lab.github.io/sns-reaction-map/
- sitemap.xml、robots.txt、OGPメタタグは設置済み
- Google Search Console に登録済み（所有権確認済み、サイトマップ送信済み）
- **構造化データ（JSON-LD）がまだない** ← 今回のタスク

### 対象ファイル（6ファイル）

すべて `docs/` ディレクトリ内：

1. `index.html` — ポータルページ
2. `ai-copyright-reaction-map.html` — 生成AIと著作権問題
3. `school-nickname-ban-reaction-map.html` — 学校でのあだ名禁止の是非
4. `takaichi-reaction-map-standard.html` — 高市文春問題
5. `henoko-student-accident-reaction-map.html` — 辺野古高校生死亡事故
6. `constitutional-amendment-reaction-map.html` — 憲法改正論議

（privacy.html と disclaimer.html にはJSON-LD不要）

### やること

#### 1. index.html に WebSite の JSON-LD を追加

`</head>` の直前に挿入：

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "name": "SNS反応まっぷ",
  "url": "https://issue-stance-lab.github.io/sns-reaction-map/",
  "description": "SNSで意見が割れる話題を投票参加型で整理。あなたはどっち？"
}
</script>
```

#### 2. 各反応マップページ（5ファイル）に Article の JSON-LD を追加

各ページの `</head>` の直前に、ページ内容に合わせた JSON-LD を挿入。例（ai-copyright）：

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "生成AIと著作権問題 SNS反応まっぷ",
  "description": "生成AIと著作権問題についてSNSの意見分布を可視化。投票参加型で今の世論を確認。",
  "url": "https://issue-stance-lab.github.io/sns-reaction-map/ai-copyright-reaction-map.html",
  "image": "https://issue-stance-lab.github.io/sns-reaction-map/ogp/default.png",
  "publisher": {
    "@type": "Organization",
    "name": "SNS反応まっぷ"
  },
  "datePublished": "2026-06-24",
  "dateModified": "2026-06-25"
}
</script>
```

他の4ページも同様に、各ページの `<title>` タグの内容を `headline` に使って作成してください。

### 成果物

- 上記6ファイルにJSON-LDを追加
- 変更したファイル一覧と内容のサマリーを出力

### 注意
- `</head>` の直前に挿入すること（`<body>` 内ではない）
- URLはすべて絶対URL（https://issue-stance-lab.github.io/sns-reaction-map/...）を使うこと
- 既存のOGPメタタグや他のコードは変更しないこと
- PRは作らず、直接mainブランチにコミットしてOK
